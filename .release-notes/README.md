# Release note format

Every independent development branch adds exactly one Markdown file in this directory.

```markdown
# Short user-facing title

## 新增功能

- Describe a feature users can see or use.

## Bug 修复

- Describe a fixed user-visible problem.
```

At least one section must contain a bullet. Omit an empty section. The text is published to GitHub Releases and displayed unchanged by the terminal and App update interfaces.
